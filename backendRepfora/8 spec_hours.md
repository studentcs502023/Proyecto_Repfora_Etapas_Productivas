## Business context

The hours module manages the monthly accumulation of hours per instructor. `HourRecord` documents are **never created manually** — they are auto-generated by `bitacoras.service.js` and `trackings.service.js` when an activity is approved or executed. This module exposes read and payment endpoints only.

**Hour sources and their SystemConfig keys:**
| Activity | HourRecord field | SystemConfig key |
|----------|-----------------|-----------------|
| Logbook review | `bitacoraHours` | `HOURS_PER_LOGBOOK_REVIEW` |
| In-person tracking | `trackingHours` | `HOURS_PER_IN_PERSON_TRACKING` |
| Virtual tracking | `trackingHours` | `HOURS_PER_VIRTUAL_TRACKING` |
| Extraordinary tracking | `extraordinaryHours` | `HOURS_PER_EXTRAORDINARY_TRACKING` |
| Certification | `certificationHours` | `HOURS_PER_CERTIFICATION` |

**Monthly limit:** `SystemConfig.MAX_MONTHLY_HOURS_INSTRUCTOR` (default 160).  
**Excess hours** are not lost — they are stored in `HourRecord.excessHours` and can be carried to the next month via `carriedOverHours`.

**Payment flow:** Admin marks hours as paid → `HourRecord.paidHours` increases, `pendingPaymentHours` decreases. Requires confirmation before applying.

## Internal helper — used by Bitacoras and Trackings services

> Define in `hours.service.js`. Called internally — not via HTTP.

```javascript
/**
 * Find or create the HourRecord for instructor/month/year.
 * Add hours to the correct field. Recalculate totals.
 * Check monthly limit and flag excess.
 * Update User.accumulatedHours and User.pendingPaymentHours.
 * Returns { record, isOverLimit, excessAmount }
 */
const addHours = async ({ instructorId, month, year, field, amount }) => {
  // field: 'bitacoraHours' | 'trackingHours' | 'certificationHours' | 'extraordinaryHours'

  const maxHours = await getConfig('MAX_MONTHLY_HOURS_INSTRUCTOR');

  let record = await HourRecord.findOne({ instructor: instructorId, month, year });
  if (!record) {
    record = new HourRecord({ instructor: instructorId, month, year });
  }

  record[field] = (record[field] || 0) + amount;
  record.totalHours = record.bitacoraHours + record.trackingHours +
                      record.certificationHours + record.extraordinaryHours;
  record.pendingPaymentHours = (record.pendingPaymentHours || 0) + amount;

  let isOverLimit = false;
  let excessAmount = 0;

  if (record.totalHours > maxHours) {
    isOverLimit = true;
    excessAmount = record.totalHours - maxHours;
    record.excessHours = excessAmount;
  }

  await record.save();

  // Update User accumulatedHours and pendingPaymentHours
  await User.findByIdAndUpdate(instructorId, {
    $inc: { accumulatedHours: amount, pendingPaymentHours: amount }
  });

  return { record, isOverLimit, excessAmount };
};

/**
 * Subtract hours when marking as paid.
 * Called by bitacoras.service and trackings.service on mark-paid.
 */
const markHoursPaid = async ({ instructorId, month, year, amount }) => {
  const record = await HourRecord.findOne({ instructor: instructorId, month, year });
  if (!record) throw new Error('HourRecord not found for payment');

  record.paidHours = (record.paidHours || 0) + amount;
  record.pendingPaymentHours = Math.max(0, (record.pendingPaymentHours || 0) - amount);
  record.lastPaymentDate = new Date();
  await record.save();

  await User.findByIdAndUpdate(instructorId, {
    $inc: { pendingPaymentHours: -amount }
  });

  return record;
};

module.exports = { addHours, markHoursPaid };
```

## Endpoints

### GET `/api/hours/instructors/:instructorId`
**Auth:** JWT — `ADMIN`, or `INSTRUCTOR` accessing their own record only  
**Description:** List all monthly HourRecords for an instructor, most recent first.

**Query params:**
- `year`: filter by year (default: current year)
- `month`: filter by specific month

**Access control:**
- `INSTRUCTOR`: `req.user.id` must equal `instructorId` → else `403`.
- `ADMIN`: no restriction.

**Response:**
```json
{
  "success": true,
  "data": {
    "instructor": { "id": "...", "fullName": "...", "email": "..." },
    "records": [
      {
        "month": 11,
        "year": 2025,
        "bitacoraHours": 10,
        "trackingHours": 6,
        "certificationHours": 2,
        "extraordinaryHours": 0,
        "totalHours": 18,
        "excessHours": 0,
        "carriedOverHours": 0,
        "paidHours": 10,
        "pendingPaymentHours": 8,

      }
    ],
    "totals": {
      "allTimeAccumulatedHours": 120,
      "allTimePendingHours": 20
    }
  }
}
```

### GET `/api/hours/instructors/:instructorId/month/:year/:month`
**Auth:** JWT — `ADMIN` or own `INSTRUCTOR`  
**Description:** Detailed breakdown for a specific month including activity-level detail.

**Service logic:**
1. Find `HourRecord` for `{ instructor: instructorId, year, month }`.
2. If not found → return empty record with all zeros (not 404 — month may not have activity yet).
3. Fetch related activity detail:
   - Bitacoras approved this month: `Bitacora.find({ instructor: instructorId, reviewedAt: { $gte: monthStart, $lte: monthEnd }, status: 'APPROVED' })`.
   - Trackings executed this month: `Tracking.find({ instructor: instructorId, executedDate: { $gte: monthStart, $lte: monthEnd }, status: { $in: ['EXECUTED', 'PAID'] } })`.
4. Return record + activity breakdown.

**Response:**
```json
{
  "success": true,
  "data": {
    "record": { ... },
    "breakdown": {
      "bitacoras": [
        { "id": "...", "apprenticeName": "...", "approvedAt": "...", "hours": 2, "isPaid": false }
      ],
      "trackings": [
        { "id": "...", "apprenticeName": "...", "type": "IN_PERSON", "executedAt": "...", "hours": 2, "isPaid": true }
      ]
    }
  }
}
```

### PATCH `/api/hours/instructors/:instructorId/month/:year/:month/mark-paid`
**Auth:** JWT — `ADMIN` only  
**Description:** Admin marks pending hours as paid for a specific month. Requires confirmation (client sends a `confirm: true` flag).

**Body:**
```json
{
  "amount": 18,
  "confirm": true
}
```

**Validations:**
- `amount`: required, number > 0.
- `confirm`: must be `true` → else `400 'Confirmation required to apply payment'`.

**Service logic (step by step):**
1. Find `HourRecord` for `{ instructor: instructorId, year, month }`. Not found → `404`.
2. If `amount > record.pendingPaymentHours` → `400 'Amount (${amount}) exceeds pending hours (${record.pendingPaymentHours})'`.
3. Update `HourRecord`:
   - `paidHours += amount`
   - `pendingPaymentHours -= amount`
   - `lastPaymentDate = now`
4. Update `User.pendingPaymentHours -= amount`.
5. Record in `AuditLog`: `action: 'HOURS_MARKED_PAID'`, `details: { instructorId, year, month, amount }`.
6. Return updated record.

---

### PATCH `/api/hours/instructors/:instructorId/carry-over`
**Auth:** JWT — `ADMIN` only  
**Description:** Transfer excess hours from one month to the next.

**Body:**
```json
{
  "fromYear": 2025,
  "fromMonth": 11,
  "toYear": 2025,
  "toMonth": 12
}
```

**Service logic:**
1. Find source `HourRecord`. If `excessHours <= 0` → `400 'No excess hours to carry over'`.
2. Find or create target `HourRecord`.
3. Move `excessHours` from source to target's `carriedOverHours`.
4. Add `carriedOverHours` amount to target's `totalHours` and `pendingPaymentHours`.
5. Set source `excessHours = 0`, `carriedOverHours = excessHours`.
6. Record in `AuditLog`.

### GET `/api/hours/instructors/:instructorId/report/:year/:month`
**Auth:** JWT — `ADMIN` or own `INSTRUCTOR`  
**Description:** Generate and return the monthly PDF report for download. If already generated and stored in Drive, return the existing URL. Otherwise generate it.

**Service logic:**
1. Find `HourRecord`. Not found → `404`.
2. If `record.reportDriveUrl` already exists → return it directly.
3. Otherwise: generate PDF with `pdfGenerator.util.js`, upload to Drive, save `reportDriveId` and `reportDriveUrl` on record.
4. Return `{ reportDriveUrl }`.
